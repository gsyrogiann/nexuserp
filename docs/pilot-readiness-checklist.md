# NexusERP Pilot Readiness Checklist

## Pilot Goal

Use this checklist before onboarding any pilot customer or controlled early
adopter onto the NexusERP release candidate.

## Product Readiness

* The customer's core use case matches the supported product scope
* Login, customers, suppliers, invoices, and reports are smoke-tested
* Known limitations for Gmail, VoIP, Telegram AI, or myDATA are documented
* Demo or starter data is available when needed

## Technical Readiness

* Required environment variables are configured
* Webhook endpoints are reachable and authenticated
* Support owner and technical owner are assigned
* Recent CI status is green
* Observability baseline is enabled and tested

## Customer Readiness

* Primary contact is identified
* Success criteria for the pilot are written down
* Expected workflows and integrations are listed
* Escalation path is agreed with the customer

## Feedback and Monitoring

* A feedback collection channel exists
* The GitHub pilot feedback template is ready to use
* Blocking issues are triaged within an agreed time window
* Incidents are logged against the pilot account or rollout wave
* Pilot learnings are fed back into release prioritization
* A support/escalation process is agreed and available

## Exit Criteria

* The customer can complete the agreed critical workflows
* No unresolved Critical or High issue blocks continued usage
* The pilot outcome is recorded as continue, expand, or stop
